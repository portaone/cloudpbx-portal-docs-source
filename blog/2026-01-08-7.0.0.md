Release 7.0.0 brings new functionality and bugfixes. 
Minimal PortaSwitch version is **MR124** because the portal requires new API to access voicemails and voicemail transcriptions. Please consider updating to **MR125** to get even more useful features.

### Enhancements:
- ICAP-2807: Allow users to filter orders by multiple statuses and payment methods
- ICAP-2810: Allow users to copy order IDorder history
- ICAP-2698: Missing exclamation mark in 'Phone book "{{phoneBookName}}" cannot be deleted while it is assigned to device profiles' error
- ICAP-2716: Use API instead of hard-coded list of models
- ICAP-2745: Allow users to play back voicemail greetings
- ICAP-2755: Allow users to access voicemails 
- ICAP-2767:  Allow users to use shortcuts to different actions applied to extensions
- ICAP-2780: Prepare a custom menu with the order history
- ICAP-2788: Allow users to delete call screening rules linked to a call screening policy
- ICAP-2797: Replace the error during fetch or hash comparison with the info message
- ICAP-2811: Allow users to leave reviews of delivered orders
- ICAP-2819: Allow to change the lowest price in the filters menu
- ICAP-2855: Remove validation of phone numbers for Advanced forwarding, Forward to SIP URI and Follow-Me
- ICAP-2868: Inability to set more than 1 email addresses for voicemail delivery
- ICAP-2759: Group phone models by a vendor
